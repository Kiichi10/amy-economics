# エイミー先生の経済学 — 設計書

## チャンネルコンセプト
- **誰が**: エイミー先生（解説者）＋ 少年（質問役）
- **何を**: 世界経済・日本経済の時事ニュースをわかりやすく解説
- **誰に**: 経済ニュースが苦手な一般視聴者
- **チャンネル名**: エイミー先生の経済学
- **参考チャンネル**: スッキリ経済学、アニメで世界経済

## 動画フォーマット

### Shorts（メイン）
- **解像度**: 1080x1920（9:16）
- **メインコンテンツ**: 1080x608（16:9）を中央配置
- **上部帯**: チャンネル名 + タイトル（約270px）
- **下部帯**: 字幕テロップ（約270px）
- **尺**: 60秒前後
- **横動画転用**: 中央16:9部分を抽出して通常動画化可能

### 映像構成
- **冒頭数枚**: 動画（AI動画生成 or アニメーション）でインパクト
- **それ以降**: 静止画パネル + Ken Burns効果
- **パネルスタイル**: 棒人間風キャラクター（スッキリ経済学準拠）
  - エイミー先生: 固定デザインの棒人間風キャラ
  - 少年: 固定デザインの棒人間風キャラ
  - モブ: 政治家・経済人のデフォルメ棒人間

### 音声
- **エイミー先生**: VOICEVOX（キャラ: TBD）
- **少年**: VOICEVOX（キャラ: TBD）
- **掛け合い形式**: 日本はこれで委員会と同方式

## コンテンツ戦略

### テーマ方針
- 参考チャンネルに倣い、時事経済ニュース中心
- 中国経済、日米関係、円安/円高、企業ニュース等
- 「身近な経済学」ネタも適宜混ぜて差別化

### 投稿戦略
- **Phase 1**: Shortsで登録者獲得（日1-2本目標）
- **Phase 2**: 登録者が増えたら長尺動画も展開
- **1トピック = 複数本**: 同テーマのShorts2本パッケージ

### 差別化ポイント
- エイミー先生＋少年の掛け合い（キャラ個性）
- 棒人間 + 固定キャラのハイブリッド
- 既存パイプラインによる高速量産

## 制作モード

### Mode A（対話型）— 現在のメインモード
- Phase単位でユーザーが確認・調整しながら進行
- i2vプロンプトを提示 → ユーザーがGrok等で生成 → 配置後に次Phase
- TinderレビューでシーンごとにSE/FX/KB設定
- 品質重視、演出を細かく調整したい時に使用

### Mode B（一括パイプライン）— 準備中
- `pipeline_shorts.py` でテーマを渡すだけで動画完成まで自動で走る
- **唯一の中断ポイント**: Phase 2画像生成後、ユーザーにi2v生成を依頼して待機
  - 将来的にはAPI連携で自動化予定
- Phase 3（Tinderレビュー）は `generate_mode_b_scene_review()` で自動生成
- 量産時、定型テーマ、お頭の判断に任せてOKな時に使用
- 環境変数: `SKIP_REVIEW=1`

### Mode A vs B 比較

| 項目 | Mode A（対話型） | Mode B（一括） |
|------|----------------|-------------|
| ユーザー介入 | 各Phase終了後に確認 | i2v生成 + タイトル選択のみ |
| Phase 3 | Tinder UIで手動レビュー | 自動SE/FX配置 |
| i2v動画 | ユーザーが自由なタイミングで生成 | Phase 2後に一度中断して依頼 |
| 品質調整 | 高い | 自動ゲート+自動修復 |
| 所要時間 | 長い（対話コスト） | 短い（自動進行） |

## 技術スタック
- **台本**: script_parser.py（日本はこれで委員会ベース）
- **音声**: VOICEVOX + script_to_audio.py
- **パネル**: Pillow（16:9で生成）
- **レンダリング**: MoviePy（9:16枠 + 16:9埋め込み + Ken Burns）
- **字幕**: audio_manifest.jsonベース
- **投稿**: youtube_uploader.py
